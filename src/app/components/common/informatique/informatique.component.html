IDE que j'ai utilisés

- Visual Studio Code: C'est l'IDE que j'utilise le plus souvent, utilisé pour programmer Python, C, HTML, CSS, JS et React JS, Angular. Il possède de nombreuses extensions et fonctionnalités d’interface que j’aime beaucoup, c’est pourquoi je l’ai utilisé tout au long du lycée jusqu’à présent.
- Eclipse L'IDE que j'utilise pour programmer Java et C++. C'est un endroit idéal avec des commandes disponibles pour créer divers packages, interfaces ou générer des fonctions. Mon choix n°1 lorsque je travaille avec Java traditionnel.
- IntelliJ IDEA: Mon choix n°1 lorsque je travaille avec Java Spring Boot. Intellij prend en charge un certain nombre de dépendances que Eclipse ou Spring Tool Suite ne prennent pas en charge.
- PHPStorm: L'IDE que j'utilise pour programmer PHP. Il a framework Laravel, Symfony, etc. Et je l'ai utilisé pour faire Dolibarr (fonction et classe).
- Spring Tool Suite, Dev C++, Notepad++: Les IDE que j'ai utilisés ne sont pas nombreux car à mon avis ils ne sont pas aussi pratiques que les IDE que j'ai listés.

API

J'ai utilisé l'API dans:
- Spring Boot: C'est là que je travaille le plus avec les API. J'ai utilisé L'API pour gérer les listes de données (vous pouvez voir dans mes projets Spring Boot pour savoir plus).
- PHP: J'ai appris à travailler avec l'API PHP pendant mon stage (ou spécifiquement PHP Dolibarr). Je dois créer les triggers concernant API pour gérer les commandes, factures et emails (très amusant).
- JavaScript: J'ai utilisé Express JS pour récuperer les données dans formulaire, les reformuler et les envoyer par nodemailer.
- Python: J'ai utilisé l'API pour obtenir des données de l'API des données (par exemple: API des stations de vélo d'une entreprise dans tous les villes pour traiter les données).

Pour tester API, Middleware Token, j'utilise PostMan. (très pratique sans avoir à tester directement sur le programme).

SQL

J'ai utilisé SQL dans:
- SQL Workbench: Ce n'est pas ma première application à utiliser SQL ou à écrire des requêtes, mais Workbench sera l'application que j'utilise le plus pour SQL. Je crée souvent des bases de données liées à des projets nécessitant des informations back-end, ou simplement des bases de données pour m'entraîner à écrire des requêtes SQL.
- PHPMyAdmin: J'ai utilisé PHPMyAdmin lorsque j'ai découvert la liaison de données entre le front et le backend (lorsque j'ai exécuté les programmes avec xamp ou wamp), la base de mon utilisation ultérieure de Workbench.
- Access: La première fois que je suis entré en contact avec SQL. Même s'il s'agissait de Jet SQL, Access m'a appris à établir des relations entre les tables, à écrire des requêtes et à créer des formulaires.

Pour créer la diagramme de relation des tables, j'utilise Quick DBD.
