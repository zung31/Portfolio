Gestion d'une librairie vendant des cahiers

  • 2023

C'est mon dernière projet Java POO à Licence 2. Je vais laisser le sujet ici.

Sujet:

On veut écrire un programme de gestion d’une librairie vendant des cahiers. Chaque cahier possède un nombre de pages. Un achat est constitué de plusieurs cahiers. Le prix d'un cahier est égal à son nombre de pages divisé par 2. L'affichage d'un achat doit indiquer un cahier par ligne avec la couleur du cahier, son nombre de pages, sa quantité (1 pour l'instant), puis une ligne indiquant le prix total de l’achat. Exemple : Jaune 20 x 1 Rouge 40 x 1 prix: 30
Il n'y a pas de nombre maximal de cahiers pour un achat. On utilisera donc la classe HashSet dans une première version et ensuite java.util.ArrayList pour stocker les cahiers.
- Écrire les classes Cahier et Achat ainsi que leurs méthodes nécessaires (constructeurs, getters, setters, affiche, toString, … )
- Vérifier que le code suivant fonctionne correctement  (le résultat doit être vrai); sinon, corriger le code : [...]
- On vend un autre article : le carnet ; un carnet possède juste une valeur de 0 à 9 indiquant un facteur de qualité. Le prix d'un carnet est égal à 3 fois son facteur de qualité.
Écrire et modifier les classes et les méthodes nécessaires. On veut permettre d'ajouter plusieurs cahiers ou plusieurs carnets à notre achat d'un coup en indiquant un paramètre supplémentaire à la méthode add, indiquant la quantité des cahiers ou des carnets que l'on veut mettre dans l’achat.

Conclusion

Peut-être que le code du programme n'est pas écrit correctement, mais il répond aux exigences de résultats.
