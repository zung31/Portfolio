Online Quiz Application

  • 2024

C'est mon projet Spring Boot le plus compliqué à ce jour car il y a une connexion entre le Front End et le Back End et j'ai dû le réaliser dans 2 folders différentes. Project Quiz-Online dispose de 2 modes : le mode Administrateur pour ajouter, modifier ou supprimer des questions ; mode Le client pourra choisir le sujet, le nombre de questions qu'il souhaite et répondre au quiz.
J'ai utilisé Spring Boot pour le Back End (port 9192), React JS (.jsx) combiné avec Bootstrap pour le Front End, MySQL WorkBench (port 3306) pour la base de données. Le site démarre sur Vite + React à l'adresse http://localhost:5173/ et l'application Spring Boot est également démarrée.
Pour permettre Front à envoyer les demandes à endpoint de Back, j'utilise CrossOrigin et l'adresse du lien du front. Et en plus, j'ai utilisé Axios pour créer un instant de axios (http://localhost:9192/api/quizzes) pour envoyer des demandes plus facilement.

Détail du projet

Quant à Back, je crée d'abord un modèle pour la question, qui contient tous les champs dont une question a besoin (Id, question, sujet, type, choix, réponse correcte). Ensuite, je crée repository, service et controller. Dans Service, il existe des fonctions simples telles que getQuestionById, updateQuestion, deleteQuestion,... pour aider les admins à pouvoir contrôler directement les questions depuis le Front. Dans Controller, plus compliqué, je devrai diviser les routeurs et déployer des GET PUT DELETE en fonction des fonctions du Service. Repository est le même que d'habitude, il suffit d'extend JPA Repository et d'ajouter les requêtes nécessaires.
Pour partie Front, j'ai utilisé React et ses dépendances (axios, bootstrap, react-router-dom,..). Il existe AddQuestion.jsx et UpdateQuestion.jsx pour récupérer les informations sur les questions et les mettre à jour dans base de données. Il y a aussi une page pour afficher tous les quiz, une page pour sélectionner le nombre de questions et le sujet pour répondre au quiz, une page quiz qui inclut la logique et la notation,.... Toutes les pages renvoient l'interface au format html (sauf les pages de service).

Avantages du projet

  • Les utilisateurs peuvent ajouter et modifier les quiz, sujets à travers Front End
  • Le programme est divisé en plusieurs routeurs, ce qui facilite la navigation
  • Belle interface

Conclusion

Comme le programme ne s'exécute que localement et par intermittence, lorsque j'éteigne le programme, toutes les données seront perdues. Et il n'y a pas de section d'authentification pour admins, donc tout le monde peut accéder à des pages modifiant les questions.
